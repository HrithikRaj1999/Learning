# Interview Readiness Rubric

Use this to score yourself honestly. A folder being present is not enough; working code, tests, and explanations count.

## Scoring

| Area | 10/10 Evidence |
|------|----------------|
| Python Core | Solves medium tasks; explains mutability, iterators, decorators, generators, exceptions, context managers, and async. |
| OOP and Design | Uses composition, protocols, dataclasses, SOLID, and patterns without overengineering. |
| Testing | Writes pytest unit/integration tests, fixtures, mocks/fakes, and dependency overrides. |
| Database | Designs schemas, indexes, transactions, migrations, and avoids N+1 query problems. |
| FastAPI | Builds APIs with Pydantic, dependency injection, auth, testing, error handling, streaming, and Docker. |
| Security | Handles JWT, password hashing, CORS, rate limits, secrets, input validation, and safe logging. |
| AI Engineering | Builds embeddings, vector search, RAG, source citations, tool calls, evals, and cost controls. |
| System Design | Explains scaling, caching, queues, observability, failure modes, and tradeoffs. |
| Portfolio | Has at least two runnable projects with READMEs, tests, Docker, and architecture notes. |

## Mock Interview Checklist

- Explain your best project architecture in 5 minutes.
- Whiteboard a URL shortener.
- Whiteboard a RAG knowledge base.
- Debug a failing FastAPI auth test.
- Refactor a tightly coupled service into testable components.
- Write a SQL query and explain the index.
- Discuss async vs sync behavior in Python APIs.
- Explain how you evaluate LLM answer quality.
